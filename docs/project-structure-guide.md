# AncientChinese-Tagger 项目目录结构与架构讲解

作为项目负责人，理解项目的目录结构是掌握全局架构、协调前后端开发以及进行汇报的基础。本项目采用典型的**前后端分离**架构，并利用 **Docker** 进行容器化编排。

---

## 📂 1. 项目根目录：全局编排层

根目录负责整个项目的配置、依赖管理以及环境隔离。

| 文件/文件夹 | 作用讲解 |
| --- | --- |
| **`backend/`** | 后端源码目录，基于 FastAPI 构建。 |
| **`frontend/`** | 前端源码目录，基于 Vue 3 + TypeScript 构建。 |
| **`compose.yaml`** | **核心编排文件**。定义了后端、前端及数据库容器如何共同启动、如何挂载代码以及如何互相通信。 |
| **`compose.prod.yaml`** | 生产环境专用编排文件，包含资源限制和端口优化。 |
| **`.env`** | **全局环境配置**。存储数据库密码、API 密钥（如 DeepSeek）等敏感及可变信息，供全项目读取。 |
| **`README.md`** | 项目门户文档，包含团队分工、核心特性和快速启动指南。 |

---

## 🐍 2. 后端架构 (`backend/`)：逻辑与数据层

后端负责处理业务逻辑、数据库交互以及 AI 模型的调度。

### 核心子目录：`backend/app/`
- **`models/`**: **数据模型层**。定义了数据库表的结构（如项目、文档、用户、标注），是数据的“骨架”。
- **`routes/`**: **接口层 (API)**。定义了前端可以调用的所有 URL 路径，负责接收请求并返回响应。
- **`services/`**: **业务逻辑层**。这是最重要的一层，负责具体的活：比如如何调用 DeepSeek API 进行解析、如何保存标注数据等。
- **`dependencies/`**: 存储通用依赖，如用户登录认证（Auth）。
- **`utils/`**: 工具类，包含结构化日志配置、文本分词算法等辅助功能。

### 关键文件：
- **`main.py`**: 后端入口，启动 FastAPI 服务，加载路由和环境变量。
- **`database.py`**: 数据库连接配置。
- **`Dockerfile`**: 描述如何将后端打包成一个 Docker 镜像。

---

## 🎨 3. 前端架构 (`frontend/`)：交互与展示层

前端负责提供沉浸式的用户界面，并将用户的操作通过接口传递给后端。

### 核心子目录：`frontend/src/`
- **`views/`**: **页面层**。对应系统中的各个功能模块：
    - `HomeView.vue`: 门户首页。
    - `ProjectsView.vue`: 项目管理列表。
    - `EditorView.vue`: **核心编辑器**，包含古文解析、实体标注和 AI 问答的全部交互。
- **`api/`**: **接口封装层**。负责与后端通信，将复杂的网络请求封装成简单的函数供页面调用。
- **`components/`**: 可复用的 UI 组件（如图标、通用的对话框等）。
- **`assets/`**: 静态资源，特别是 `styles/` 目录下定义的护眼宣纸美学 CSS 变量。
- **`router/`**: 定义了页面之间的跳转逻辑。

### 关键文件：
- **`App.vue`**: 前端根组件，定义了全局布局。
- **`main.ts`**: 前端入口文件。
- **`vite.config.ts`**: 开发服务器与构建配置。

---

## 🔗 4. 核心模块关系图

理解各部分是如何“串”起来的：

1.  **数据流向**：用户在 `frontend/views/EditorView` 进行操作 -> 调用 `frontend/api/` -> 发送请求给 `backend/routes/` -> 由 `backend/services/` 处理逻辑（期间可能调用 DeepSeek AI 或 `backend/models/` 存取数据库） -> 返回结果给前端展示。
2.  **配置流向**：根目录 `.env` 的配置被 `compose.yaml` 读取 -> 注入到容器环境变量 -> 后端 `main.py` 加载并生效。
3.  **视觉统一**：`frontend/assets/styles/variables.css` 定义的色彩系统贯穿所有 `views/` 页面，确保了古风学术美感的统一。

## 💡 汇报建议
在汇报时，可以重点强调：
- **解耦性**：前后端完全分离，通过 `api/` 目录进行标准协议通信。
- **安全性**：通过 `dependencies/auth.py` 确保数据访问权限，通过 `.env` 隔离敏感配置。
- **可维护性**：核心逻辑集中在 `services/`，UI 状态集中在 `views/`，结构清晰，易于扩展功能。
