## 遇到的问题和解决
1. 问题：FastAPI 的默认日志是文本格式，无法直接满足结构化要求。
   解决：通过继承 `logging.Formatter` 编写了自定义的 JSON 格式化器，并将其挂载到 root logger 上。
2. 问题：需要在不入侵业务逻辑的情况下统计所有接口耗时。
   解决：使用了 FastAPI 的中间件（Middleware）机制，在请求前记录时间，请求后计算差值并记录日志。
